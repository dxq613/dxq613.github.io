简介

列表的应用范围很广，所有的使用数组作为数据源的控件都可以使用列表，BUI中为了针对不同的列表进行针对性的处理，我们将列表分为2类：

  1. 列表项是控件的列表：菜单、Tab标签、Toolbar以及进度条都是这一类控件
  2. 列表项不是控件，仅仅是一个Object对应一个DOM：简单列表、表格、树就是这一类控件

列表的功能

在具体介绍列表前，先来讨论一下列表需要实现的功能：

  1. 展示数组数据
  2. 选中数据：单选、多选
  3. 呈现数据的状态
  4. 增删改数据项
  5. 支持键盘事件

列表和简单列表

菜单、工具栏一类的控件，列表项是子控件的原因在于这些控件非常复杂，列表项也可能嵌套一些其他控件，例如一个菜单项中可以有子菜单；一个Tab标签项里面可能有表单、表格等控件

而简单列表、表格、树这一类控件是用于展示、操作数据，所以关注点在数据集合上，所以不将列表项实现为子控件

列表和简单列表从概念上来说没有任何差别，数据的展示、操作、选中都完全一致，只是具体的实现上有所差异，两者的抽象接口都定义在BUI.Component.UIBase.List中，选中定义在BUI.Component.UIBase.Selection中

列表和简单列表最大的差异在于列表项，对于列表而言所有的列表项时一个个的Controller类，所有的属性通过get,set访问和设置，而简单列表的列表项是一个个的Object，可以直接操作Object，但是对应的DOM状态需要函数同步。

列表操作

这里仅介绍列表的增删改，列表的选中在后面的菜单、Selection章节里介绍，所有的列表都支持下面的操作：

  1. addItem( item ),addItemAt( item, index ) : 添加选项
  2. addItems( items ),addItemsAt( items,index ) ： 批量添加选项
  3. removeItem(item),removeItemAt(index) : 移除选项
  4. removeItems(items) : 批量移除选项
  5. updateItem( item ) ：更新选项

列表项的查找

对列表项进行操作就牵扯到列表项的查找问题，BUI的列表提供了下面几个查找函数：

  1. indexOfItem( item ) : 查找选项的索引值
  2. getItems( ) ： 获取所有的选项
  3. getFirstItem( ),getLastItem( ) : 获取第一个、最后一个选项
  4. getItemAt( index ) ： 根据索引查找选项
  5. getItem( id ): 根据选项的id获取选项
  6. getItemCount( ) : 获取所有选项的数量

列表的事件

由于列表项是Controller类，所有包含Controller的所有事件，但是简单列表的事件需要重新定义，所以2者共同的事件只有一个（除选中引起的事件）：itemclick,具体每个控件的事件会在后面的章节里一一介绍
